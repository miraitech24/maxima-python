# 課題：デジタル脳の「無意識」を可視化する

## 1. 課題の背景：マインドアップロードとは

マインドアップロード（脳のデジタル化）とは、生身の脳で行われている「電気信号のやり取り」を、コンピュータ上の「計算」として再現する試みです 1。本課題では、その最小単位である「ニューロン（神経細胞）」が、外部からの刺激を受けて反応するプロセスを扱います。

## 2. 「電気が溜まる」現象の意味

デジタル脳において、グラフの横軸（時間）とともに電圧が上がっていく様子は、脳が**無意識に情報を蓄積している状態**を表します。

- **自動的な反応**: 心臓の鼓動のように、デジタル化された細胞も刺激が入れば物理法則に従って自動的に電気を溜め始めます。

- **無意識の計算**: 電気が溜まっている最中は、まだ「意識」にのぼる前の情報処理（準備段階）です。

- **時間の意味**: 横軸が進むことは、デジタル世界で「時計の針が進む」ことであり、それと同時に情報が蓄積されていくプロセスそのものです。

- **数式の表示**: Maximaを用いて、脳の活動を定義する数式を正しく表示すること 。

- **シミュレーションの実行**: Pythonを用いて、時間が経つにつれて「無意識」に電気が溜まっていく様子をグラフ化すること 。

---

## 3. 課題概要

マインドアップロードの基礎技術となる「単一ニューロンの活動電位」を解析します。神経膜の等価回路モデルである**Hodgkin-Huxley方程式**の一部を簡略化したモデルを用い、平衡点の解析と刺激応答シミュレーションを、MaximaとPythonを連携させて実行します。

## 4. 数学モデル（Maxima担当）

神経膜の電位変化 $V$ は、以下の微分方程式で記述されます。

$C_m \frac{dV}{dt} = I_{ext} - g_L (V - E_L) - g_{Na} m^3 h (V - E_{Na}) - g_K n^4 (V - E_K)$

### Maximaでの処理内容

- 定常状態における平衡点 $V_{rest}$ の代数的導出。

- 各イオンチャネルのゲート変数（$m, h, n$）の転移速度定数に関する数式の簡略化。

- 結果を `params.py` に書き出す処理。

---

## 5. 数値シミュレーション（Python担当）

Maximaで算出した平衡点を初期値として読み込み、外部刺激 $I_{ext}$ を与えた際の膜電位の時系列変化を計算します。

### Pythonでの処理内容

- `params.py` のインポート。

- Runge-Kutta法による微分方程式の数値解法。

- プロットによる可視化。

# 課題2：デジタル脳における超高速リレーと対数的飽和の連成シミュレーション

## 1. 課題の背景と目的

本プロジェクトは、マインドアップロード（脳のデジタル化）における「意識の発生」を数理モデル化する試みです。

文字情報のような逐次的な処理と、画像情報のような並列的な処理の違いを、**「\mu s（マイクロ秒）単位のデジタル・リレー」と「ms（ミリ秒）単位のアナログ的飽和」**という2つの視点から可視化します。

## 2. 理論：最後（#5）は（なだらかな曲線）へ

本シミュレーションの核心は、**「個別のニューロンの激しい火花が、ネットワークの終着点では安定した一つの認識（曲線）へと収束する」**というプロセスにあります。

- 幾何級数的なリレー（#1〜#4）:
  
  情報はデジタル的なスパイクとして爆発的に伝播します。これは対数軸で見ると垂直な立ち上がりとして表現されます。

- 対数的な飽和（#5）:
  
  リレーの最後（#5）のニューロンは、ネットワーク全体の信号を集約します。物理的限界（サチュレーション）により、その挙動は**初期モデルで提示された「なだらかな上昇曲線」**へと相似的に回帰します。

## 3. ツール・カップリングの役割

本課題は Maxima と Python を連携（カップリング）させることで、論理と描写を両立させています。

| **ツール**    | **担当フェーズ**    | **デジタル脳での意味**                                     |
| ---------- | ------------- | ------------------------------------------------- |
| **Maxima** | **理論・代数解析**   | 脳が「納得（サチュレート）」する物理的な境界条件を数式で厳密に定義します。             |
| **Python** | **動的描写・並列処理** | Maximaの解を元に、\mu s スケールでの高速リレーが安定した曲線へ移行する様を描写します。 |

## 4. プログラム構成と連携

1. **`analysis.mac` (Maxima)**:
   
   - 飽和レベル V_{inf} と収束の速さ \tau を算出します。
   
   - `display2d: false` 設定により、Pythonが直接読み込める形式で数値を書き出します。

2. **`params.py` (中間ファイル)**:
   
   - MaximaからPythonへ物理定数を渡す「情報の橋渡し」を担います。

3. **`simulation.py` (Python)**:
   
   - 縦軸を対数（Log Scale）に設定し、#1から#5までのリレーを可視化します。
   
   - 最終的に、**#5の曲線が初期提示グラフと一致する**ことを確認します。
